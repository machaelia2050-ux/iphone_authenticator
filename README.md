# iPhone Authenticator System

A Flask web application that verifies whether a phone is a genuine iPhone using multiple automated checks (image AI analysis, IMEI verification, model number validation, and barcode scanning)  producing a combined authenticity score out of 100.

---

## What It Does

Fake iPhones are a real problem in second-hand markets. This system helps buyers and resellers verify a phone before completing a purchase. It runs six independent checks and combines the results into a final verdict:

| # | Check | Points |
|---|-------|--------|
| 1 | Image AI Analyser (MobileNetV2) | up to 20 |
| 2 | Barcode Scanner (EAN-13 / UPC-A) | up to 5 |
| 3 | IMEI Comparator (box / settings / dial) | up to 5 |
| 4 | IMEI Online (TAC database lookup) | up to 30 |
| 5 | IMEI Offline (Luhn checksum + TAC prefix) | up to 15 |
| 6 | Model Number Validator | up to 25 |
| | **Total** | **100** |

A score of 75 or above is considered **Likely Original**. Below 60 is **Likely Fake**. Any mandatory check that fails hard (image, IMEI, or model number) triggers an immediate fake verdict regardless of the total score.

---

## Features

**User App**
- Upload a back photo of the phone - the AI crops the camera and logo regions separately and classifies both
- Camera barcode scanner (powered by QuaggaJS) with manual fallback entry
- Three-way IMEI comparison (box label, Settings, `*#06#`)
- TAC-based IMEI lookup covering all iPhone 11–17 models across all regions
- Model number lookup against a 100+ entry Apple database (iPhone 11–17, all variants, all regions)
- Responsive design - works on desktop and mobile
- Buyer's advice section with physical inspection tips
- Feedback / challenge reporting

**Admin Panel** (`/admin`)
- Password-protected dashboard
- Every check request logged with timestamp, IP address, and geolocation
- Google Maps link for each user location
- Score badges, verdict filters, and full-text search
- Per-row and bulk delete

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Collect training images

Put back-of-phone photos into the two folders:

```
data/raw/real/    ← genuine iPhone back photos (aim for 200+)
data/raw/fake/    ← Android or fake phone back photos (aim for 200+)
```

Photos should clearly show the camera module and Apple logo. Remove phone cases. Any common image format is supported (JPG, PNG, WEBP, HEIC, AVIF, etc.).

### 3. Prepare the dataset

```bash
python prepare_data.py
```

This crops each photo into two regions, the camera bump (top 45%) and the logo area (40–78%), and saves them to `data/processed/`. Your originals are never modified.

### 4. Train the classifier

```bash
python train.py
```

Fine-tunes a pretrained MobileNetV2 on your cropped images. Training runs for up to 40 epochs with early stopping. The best checkpoint is saved automatically to `models/iphone_classifier.pth`.

Typical output on 200 real + 200 fake photos: **val_acc ≈ 90–96%**

### 5. Run the app

```bash
python app.py
```

Open [http://127.0.0.1:5000](http://127.0.0.1:5000) in your browser.  
Admin panel: [http://127.0.0.1:5000/admin](http://127.0.0.1:5000/admin)

## How the Image Check Works

Rather than classifying the whole photo (which would be thrown off by backgrounds, hands, and cases), the system:

1. Crops the **camera module region** (top 45% of the image)
2. Crops the **logo region** (40–78% of the image height)
3. Runs the classifier independently on each crop
4. Averages the two probabilities

This makes the model focus on iPhone-specific hardware features rather than overall phone shape. A Samsung Galaxy fails because its camera layout and back design differ structurally from an iPhone's.

---

## Evaluating and Tuning the Model

After training, run the threshold checker to see per-image results:

```bash
python threshold_check.py
```

This shows the iPhone probability for every image in `data/raw/real/` and `data/raw/fake/`, overall accuracy, and tuning recommendations. If too many real iPhones are being rejected, lower `IPHONE_THRESHOLD`. If fakes are passing, raise it or add more fake training images.

To test a single image from the command line:

```bash
python predict.py --image path/to/photo.jpg
```

## IMEI and Model Number Coverage

**IMEI TAC database** - covers all Apple iPhone models from iPhone 11 through iPhone 17 across all regions (USA, Global, China, Japan, Korea). The check matches the first 8 digits of the IMEI against known Apple TAC codes.

**Model number database** - 100+ entries covering every regional variant of iPhone 11–17 (e.g. `A2649` = iPhone 14 USA, `A2882` = iPhone 14 Global). Unknown model numbers prompt the user to verify manually at imei.info before the check continues.

## Verdict Logic

| Score | Verdict |
|-------|---------|
| 90–100 | 🔥 Very Likely Original |
| 75–89 | ✅ Likely Original |
| 60–74 | ⚠ Uncertain, inspect manually |
| 0–59 | ❌ Likely Fake |

Any hard failure on the image check, IMEI online check, or model number check overrides the score and gives an immediate **Fake** verdict.

## Deploying with Gunicorn

```bash
gunicorn -w 2 -b 0.0.0.0:5000 app:app
```

For production, put Nginx in front and set a proper `SECRET_KEY` and `ADMIN_PASSWORD` in environment variables rather than in the source file.
